JTerm Terminal Emulator Technical Manual

 

JTerm is a simple and flexible terminal emulator. JTerm is written in the Java language so it can be used on any system supporting Java2.

 

JTerm has been designed from the ground-up to be easy to understand and enhance. What follows are the design specifications and the technical description of JTerm.

 

JTerm is covered by the GNU public license, which means it is free and all the code is freely available.

Program Description

 

JTerm is a Terminal Emulation program. A Terminal Emulation program emulates the functions of a character-based terminal, such as the Data General D210.

 

JTerm design is based on the MVC model. MVC stands for Model-View-Controller. MVC is a simple and widely used architecture for designing visual components. For more on MVC click here.

 

JTerm was designed to be extremely flexible. The user can change each module:

  • Models – The Terminal Emulation Type can be easily changed. The currently supported emulations are:
    • Data General D210
  • Views – The window which you view the screen can also be easily changed. Currently supported:
    • Java Application Window
    • Java Applet Window
    • HTML Window
  • Controls – The method of connecting to the host computer can be selected and configured by the user:
    • Serial connection
    • TCP/IP (to a server with a serial connection to the host)

 

Design Goals

JTerm was designed with the following goals in mind:

  • A Simple Basic Design that is easy to understand.
  • Modules that can be easily enhanced for added functionality.
  • A Modular design that can be easily changed without affecting other sections of the program.

 

Design Summary

 

The basic design for JTerm is very simple. It consists of the interfaces for the Model, View, and Control. A programmer writing an enhancement for JTerm has the flexibility to write the code from scratch, or use one of the included modules as a base.

 

UML Interface Diagram

 

The four main sections of code for JTerm are the:

  • Models - Emulations
  • Views - Screens
  • Controls – Physical links to the host
  • Property Functions – These are the screens and classes that hold the parameters for the Models, Views, and Controls.
