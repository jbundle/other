Style Reference

These are the notes that explain my coding style.

 

First, I try whenever possible to conform to the javasoft coding style guidelines with the following exceptions:

 

I use Hungarian notation for my variables. I still find it helpful to know a variables scope and type without searching the code.

 

I never use an empty constructor to insatiate an object. If a class does not require a value for the constructor, then I call new with a null value. I rarely put code in the constructor; I put it in a method called “init” with identical variable list as the constructor. This makes it easy to create and initialize a class from only the class name.

 

I Usually have a “free” method that releases any resources that the object has acquired.
