JTerm Terminal Emulator User Manual

 

JTerm is a simple and flexible terminal emulator. JTerm is written in the Java language so it can be used on any system supporting Java2.

 

Currently JTerm supports the following emulations:

  • DataGeneral D210

 

JTerm is covered by the GNU public license, which means it is free and all the code is freely available. If you are interested in the technical documentation, take a look at the JTerm technical documentation. JTerm has been designed from the ground-up to be easy to understand and enhance.

Program Description

 

JTerm is a Terminal Emulation program. A Terminal Emulation program emulates the functions of a character-based terminal, such as the Data General D210.

 

JTerm was designed to be extremely flexible. The user can change each module:

  • Models – The Terminal Emulation Type can be easily changed. The currently supported emulations are:
    • Data General D210
  • Views – The window which you view the screen can also be easily changed. Currently supported:
    • Java Application Window
    • Java Applet Window
    • HTML Window
  • Controls – The method of connecting to the host computer can be selected and configured by the user:
    • Serial connection
    • TCP/IP (to a server with a serial connection to the host)

JTerm Menus

 

This JTerm menus are self-explanatory, except for the Properties menu:

 

Properties menu:

The properties menu is used to modify the Model, View, or Control of the JTerm program.

 

Properties screen

The Model is the terminal emulation to use.

The View is how the terminal should be displayed.

The Control describes the physical connection to the host.

 

The Models that are currently available are:

  • Data General D210

 

The Views that are currently available are:

  • Screen – This displays the information in a java application or applet window.

 

The Controls that are currently available are:

  • Serial – Connects through the computer’s serial port.

Note: The Java Serial Communication must be installed on your computer for this to work. If you don’t have a computer geek to help you, follow these simple instructions.

  • RMI – Connects to the host over a TCP/IP network.

Note: You must have the JTerm Server running and properly configured to use this option.

  • Loopback – Used for testing (Any character typed on the keyboard will be sent to the screen for display)
